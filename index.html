Ahora que tu repositorio está creado y ya has hecho tu primer commit, el proceso para hacer nuevos cambios y actualizarlos en GitHub es un ciclo repetitivo de tres pasos principales ModificarCrear archivos Realiza los cambios que necesites en tus archivos de código (por ejemplo, actualiza game.js, modifica index.html, añade nuevos archivos, etc.). Preparar los cambios para un commit (git add) Una vez que estés satisfecho con tus modificaciones, le indicas a Git qué cambios quieres incluir en tu próximo punto de guardado. Realizar el commit (git commit) Creas un nuevo punto de guardado en el historial de tu proyecto, con un mensaje que describe lo que cambiaste. Subir los cambios a GitHub (git push) Envías tus nuevos commits desde tu repositorio local al repositorio remoto en GitHub. Aquí te explico los comandos Ciclo de trabajo para nuevos commits y actualizaciones en GitHub Cuando quieras hacer nuevos cambios y subirlos, seguirás estos pasos en tu terminal (en la carpeta de tu proyecto) Paso 1 Realiza tus cambios en los archivos. Por ejemplo, abres jsgame.js y haces mejoras, o creas un nuevo archivo CSS. Paso 2 Prepara tus cambios (git add). Después de modificar o crear archivos, necesitas decirle a Git que los prepare para el próximo commit. Para añadir todos los cambios (recomendado para empezar) Bash git add . El punto (.) indica que se añadan todos los archivos nuevos, modificados o eliminados en el directorio actual y sus subdirectorios. Para añadir archivos específicos Bash git add nombre_del_archivo.js git add carpetaotro_archivo.html Esto es útil si solo quieres incluir ciertos cambios en un commit. Paso 3 Realiza un nuevo commit (git commit). Una vez que los cambios están preparados, los guardas en el historial de tu repositorio con un mensaje descriptivo. Bash git commit -m Mensaje descriptivo de tus cambios aquí git commit El comando para crear un commit. -m Mensaje... El flag -m te permite escribir un mensaje corto directamente en la línea de comandos. Es crucial que este mensaje describa claramente qué hiciste en este commit (por ejemplo, Añadir control táctil, Corregir bug de colisión, Implementar puntuación más alta local). Paso 4 Sube tus cambios a GitHub (git push). Este comando envía tus nuevos commits desde tu repositorio local a tu repositorio remoto en GitHub. Bash git push origin master git push El comando para enviar cambios. origin Es el nombre corto que se le dio a tu repositorio remoto de GitHub cuando lo enlazaste por primera vez. master Es el nombre de la rama local desde la que estás empujando (y que quieres actualizar en GitHub). Resumen del proceso en una línea de tiempo Editas game.js. git add . (preparas los cambios). git commit -m Mejorar la lógica del movimiento de la serpiente (guardas los cambios). git push origin master (envías los cambios a GitHub). Cada vez que quieras actualizar tu proyecto en GitHub, repetirás los pasos 2, 3 y 4.
